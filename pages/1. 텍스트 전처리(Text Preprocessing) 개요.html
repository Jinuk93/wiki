텍스트 전처리(Text Preprocessing)란?

컴퓨터가 텍스트를 이해하기 위한 첫 번째 단계

1부 : 텍스트 전처리의 기본 단계

가장 보편적으로 사용되는 네 가지 기본 전처리 단계를 통해 텍스트를
정제하는 방법을 알아봅니다.

1

정제 (Cleaning)

텍스트에서 분석에 불필요한 노이즈를 제거하는 과정입니다.

  • HTML 태그 제거
    :     웹에서 수집한 데이터의 <a>, <br> 같은 태그들을 없애줍니다.
  • 특수 문자 및 부호 제거
    :     문장 분석에 불필요한 #, @, $ 같은 기호나 문장 부호를 제거합니다.
  • 대소문자 통일 (주로 영어)
    :     'Apple'과 'apple'을 컴퓨터가 같은 단어로 인식하도록 모두 소문자로 바꿔줍니다.
2

토큰화 (Tokenization)

정제된 텍스트를 의미 있는 최소 단위인 토큰(token)으로 분절하는 과정입니다.

  • 단어 토큰화
    :     문장을 공백이나 문장 부호를 기준으로 단어 단위로 나눕니다.
  • 문장 토큰화
    :     긴 글(문서)을 마침표(.), 물음표(?) 등을 기준으로 문장 단위로 나눌 수도 있습니다.
3

불용어 제거
(Stopword Removal)

의미 분석에 큰 도움이 되지 않는, 문법적 기능을 하는 단어들을 제거합니다.

  • 불용어(Stopwords)란?
    : 은, 는, 이, 가 (한국어), a, the, is (영어) 처럼 의미는 거의 없지만 자주 등장하는 단어들을 말합니다.
4

단어 형태 통일

같은 의미를 가진 여러 형태의 단어들을 하나의 기본 형태로 통일하는 작업입니다.

  • 어간 추출 (Stemming)
    :     단어의 뒷부분을 규칙에 따라 잘라 원형을 찾습니다. (예: studies, studying → studi)
  • 표제어 추출 (Lemmatization)
    :     단어의 품사 정보를 고려해 사전적인 기본형을 찾습니다. (예: studies, studying → study)

2부 : Task와 모델에 따른 전처리 전략

모든 상황에 맞는 완벽한 전처리는 없습니다. 목표와 도구에 따라 전략은 달라져야 합니다.

나의 목표는 무엇인가?

스팸/정상 메일 분류인가? (핵심 단어가 중요)
사용자 질문에 답을 검색하는 챗봇인가? (정보의 정확성이 중요)
긴 글을 요약하거나 다른 언어로 번역하는 것인가? (원문의 의미와 뉘앙스가 중요)

왜 모델마다 전처리 방법이 달라야 할까?

핵심은 모델이 텍스트를 이해하는 방식이 근본적으로 다르기 때문입니다.

전통 모델 : TF-IDF

: 단어를 '낱개의 구슬'처럼 봅니다. 단어의 순서(문맥)는 무시하고, 문서 내 출현 빈도를 중심으로 특징을 추출합니다.

실행
: 기본 전처리 4단계를 적극적으로 사용하여 '신호'가 되는 핵심 단어만 남기고 '노이즈'를 최대한 제거합니다.

최신 모델 : BERT, GPT

: 단어를 '문장의 일부'로 봅니다. 단어의 순서, 주변 단어와의 관계 속에서 동적인 의미를 파악합니다.

실행
: 문맥 정보를 최대한 보존하기 위해 전처리를 최소화합니다. 조사, 관사 등도 중요한 단서로 활용됩니다.
분류 (Classification)

전략적 전처리

목표 모델에 맞춰 전처리 강도를 조절합니다. 전통 모델은 강하게, 최신 모델은 부드럽게.

  • TF-IDF
    :     불용어 제거, 단어 형태 통일로 핵심 단어의 통계적 중요도 극대화
  • Transformer
    :     문맥 파악에 중요한 조사, 시제 정보 등을 보존
검색 / RAG

보존적 전처리

정보의 정확성이 생명! 검색의 핵심이 될 단서를 훼손하지 않는 것이 가장 중요합니다.

  • 고유명사 (예: Python)
  • 전문 용어, URL, 코드 블록
  • 위와 같은 핵심 정보를 보존하는 방향으로 클리닝 규칙을 설계합니다.
생성 (Generation)

최소한의 전처리

원문의 의미와 뉘앙스를 조금이라도 바꾸면 결과물의 품질이 크게 저하될 수 있습니다.

  • "I am not happy"에서 'not'을 제거하면 의미가 반전됩니다.
  • 마침표, 쉼표 등은 문장의 리듬과 구조를 형성하는 중요 요소입니다.
  • 오타나 깨진 문자 정리 외에는 원본을 최대한 유지합니다.
