미니배치(Minibatch) 생성과정

AI 모델을 초고속으로 학습시키는 GPU는 반듯하게 정렬된 숫자 블록(텐서)만 이해하고 처리할 수 있습니다.
하지만 우리가 다루는 텍스트는 문장마다 길이가 제각각인, 불규칙한 데이터의 연속이죠.

이 둘 사이의 간극을 메우는 기술이 바로 미니배치 생성 과정입니다.
이 과정은 크게 두 단계의 문제를 해결하는 과정으로 나눌 수 있습니다.

1

1단계 : 글자를 숫자로 바꾸기 (Indexing)

가장 먼저 해결할 문제는 "글자를 어떻게 숫자로 바꾸는가?"입니다.
GPU는 '사과'라는 글자를 직접 이해하지 못합니다.
그래서 우리는 일종의 '단어-숫자 번역사전(Vocabulary)'을 만듭니다.

1. 단어장 생성

텍스트에 있는 모든 고유 단어(토큰)를 모아 번호표를 붙여줍니다.

{'': 0, '나': 1, '는': 2, '사과': 3, '를': 4, ...}

2. 인덱싱(Indexing)

이 단어장을 기준으로 문장을 숫자들의 나열(리스트)로 바꿉니다.

['나', '는', '사과', '를'] → [1, 2, 3, 4]

이 과정을 통해, 수만 개의 0과 하나의 1로 이루어진 거대한 원-핫 벡터를 통째로 저장하는 대신,'1'이 위치한 단 하나의 '인덱스'만 기억하게 됩니다.
덕분에 메모리 문제가 해결되고 텍스트는 훨씬 효율적인 숫자 데이터로 압축됩니다.

2

2단계 : 길이를 통일하기 (Padding)

이제 모든 문장이 숫자 리스트로 바뀌었지만, 아직 문제가 남아있습니다.
바로 "문장마다 길이가 다르다"는 모양 문제입니다.

[1, 2, 3, 4] (길이 4)
[1, 5, 6] (길이 3)

길이가 다른 숫자 리스트들은 반듯한 직사각형 형태로 쌓을 수 없습니다. 마치 높이가 다른 책들을 쌓으면 삐뚤빼뚤해지는 것과 같죠. GPU는 이렇게 모양이 제각각인 데이터를 한 번에 처리하지 못합니다.

이때 사용하는 기술이 바로 패딩(Padding)입니다.

패딩(Padding)이란?

한 묶음(미니배치) 내에서 가장 긴 문장을 기준으로,
나머지 짧은 문장들의 뒷부분을 의미 없는 특별 토큰인 <pad>의 인덱스(여기서는 0)로 채워
길이를 똑같이 맞추는 작업입니다.

적용 예시 (가장 긴 문장 길이 = 4)

[1, 2, 3, 4] (길이 4) → [1, 2, 3, 4] (변화 없음)
[1, 5, 6] (길이 3) → [1, 5, 6, 0] (길이가 4로 맞춰짐)

이제 길이가 제각각이던 숫자 리스트들이 모두 같은 길이를 가진 숫자 리스트로 바뀌었습니다.

3

3단계 : 완벽한 숫자 블록, 텐서(Tensor)의 완성

드디어 마지막 단계입니다. 패딩 덕분에 우리는 이제 길이가 똑같이 맞춰진 여러 개의 숫자 리스트를 갖게 되었습니다. 이제 이 리스트들을 차곡차곡 쌓기만 하면 됩니다.

[1, 2, 3, 4]
[1, 5, 6, 0]
[7, 8, 3, 0]
... (이하 32개 문장)

이렇게 반듯하게 쌓인 숫자들의 2차원 행렬(Grid)이
GPU가 원하는 최종 데이터 형태인 텐서(Tensor)입니다.

텐서(Tensor)란?

AI 모델링에서 사용되는 다차원 숫자 배열을 의미합니다.
지금 우리가 만든 것은 (미니배치 크기 × 문장 길이) 형태의 2차원 텐서입니다. (예: 32 × 4)

결론

결론적으로 패딩은 제각각의 길이로 벡터화된 길이에 ‘0’으로 채워줌으로서,
GPU에서 처리할 수 있는 균일한 모양의 텐서로 만들어 ‘하나의 미니배치(Minibatch)’가 만들게 됩니다.

미니배치
: 학습 과정에서의 역할을 부르는 이름 (예: 32개의 문장 묶음)

텐서
: 그 데이터 묶음의 실제 모양(shape)을 부르는 이름 (예: [32, 100] 모양의 숫자 행렬)

미니배치 살펴보기

미니배치(Minibatch)는 전체 학습 데이터셋을 여러 개의 작은 묶음으로 나눈 것입니다.
학습 시에 데이터 전체가 아닌, 이 미니배치 단위로 데이터를 입력받아 가중치를 업데이트합니다.

미니배치의 역할과 목적

미니배치를 사용하는 이유는 크게 계산 효율성과 학습 안정성 두 가지입니다.

1. 계산 효율성 (Computational Efficiency) :

여러 개의 데이터 샘플(벡터)을 하나의 미니배치(행렬/텐서)로 묶어서 전달하면, GPU는 병렬 처리를 통해 훨씬 빠른 속도로 연산을 수행할 수 있습니다.

2. 학습 안정성 (Training Stability) :

모델은 데이터의 예측 오차를 기반으로
가중치를 얼마나 수정할지(그래디언트, Gradient)를 계산하며 학습합니다.

  • 데이터 1개만 사용
    : 1개의 데이터로 계산된 기울기는 불안정하고 노이즈가 심해, 학습 방향이 크게 흔들릴 수 있습니다.
  • 전체 데이터 사용
    : 전체 데이터로 계산된 기울기는 가장 정확하지만, 데이터가 클 경우 많은 시간과 메모리가 소요됩니다.
  • 미니배치 사용
    : 전체 데이터의 기울기에 대한 좋은 근사치(approximation) 역할로서,
    전체 데이터 사용 보다 속도가 훨씬 빠릅니다. 이는 모델이 효율적으로 최적의 가중치를 찾습니다.
